# v0.1.1 - 2026-03-20

## 版本主题
安卓端“论文索引增强”第二阶段落地版，重点补齐双语摘要能力与旧记录补生成路径。

本版本在 `v0.1.0` 的基础上继续完善：

- 中文优先展示
- 英文保真保留
- 详情页中英切换
- 旧记录重新生成双语摘要

---

## 本次版本完成内容

### 1. 双语摘要字段落地
论文元数据当前额外支持：

- `summary_zh`
- `summary_en`

它们与现有字段关系如下：

- `summary_short`：移动端默认展示的中文极短摘要
- `summary_zh`：中文摘要，适合移动端快速理解
- `summary_en`：英文摘要，保留学术原味
- `summary`：兼容字段，继续保留

### 2. 详情页默认中文，保留英文切换
论文详情页当前行为：

- 默认优先显示中文摘要
- 如果同时存在 `summary_zh` 和 `summary_en`，显示 `中文 / English` 切换
- 切换偏好会在本地记忆，下次进入继续沿用

### 3. 列表页双语标识
论文列表卡片当前如果具备双语摘要，会显示：

- `EN`
- `支持中英切换`

这样用户在列表阶段就能判断这条记录是否支持中英切换。

### 4. 旧记录支持手动重新生成双语摘要
论文详情页右上角 `...` 菜单当前新增：

- `重新生成双语摘要`

这条链路会：

- 读取当前条目的标题和正文
- 调用 AI 重新生成：
  - `summary_zh`
  - `summary_en`
  - `summary_short`
- 写回 `meta_json`

适用于：

- 旧记录没有双语摘要
- 想重新生成更好的中英摘要

### 5. AI 输出约束增强
当前 prompt 已明确要求：

- `summary_short` 必须为中文且不超过两句
- `summary_zh` 用于移动端展示
- `summary_en` 保留研究语感和术语

因此当前产品策略是：

- 手机端默认中文
- 英文保真保留
- 不再强制二选一

## 当前版本的数据契约变化

相较于 `v0.1.0`，论文 `meta_json` 结构补充：

```json
{
  "summary_short": "中文极短摘要",
  "summary_zh": "中文摘要",
  "summary_en": "英文摘要"
}
```

当前 Web / 后端如果要消费新的摘要能力，建议优先级：

1. `summary_short`
2. `summary_zh`
3. `summary_en`
4. `summary`

---

## 面向 Web / 后端开发者的新增说明

### 1. 中文默认展示策略
当前安卓端推荐展示优先级：

- 列表页：`summary_short`
- 详情页默认：`summary_zh`
- 详情页英文切换：`summary_en`

### 2. 英文仍然保留
产品上不再把英文摘要删掉或替换掉，而是：

- 中文做入口展示
- 英文做保真内容

这对 Web / 桌面端也成立。

### 3. 旧记录并不天然有双语字段
`v0.1.1` 之前创建的记录，可能没有：

- `summary_zh`
- `summary_en`

当前安卓端已经提供人工触发的重新生成入口，但 Web / 后端如果要做批量补全，仍需单独开发批处理逻辑。
