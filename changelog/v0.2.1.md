# v0.2.1 - 2026-03-21

## 版本主题
安卓端“链接先分类入库 + 后台解析状态可视化”体验升级版本。

本版本聚焦的重点不是提升解析模型能力本身，而是修复“用户输入链接后长时间无反馈、必须等待解析完成才能看到条目”的交互问题。目标是把链接采集链路改造成“先有结果，再慢慢变好”的模式。

---

## 本次版本完成内容

### 1. 链接采集流程改为“先分类，再后台解析”
原先流程是：

- 输入链接
- 等待网页抓取
- 等待 AI 解析
- 最后再选择分类并入库

新流程改为：

- 输入链接
- 立即选择分类
- 先创建一个 `processing` 占位条目
- 条目立刻进入对应分类
- 后台继续抓取网页与 AI 解析
- 完成后自动回填条目内容

这一步直接解决了“用户必须盯着一个黑盒等待结果”的问题。

### 2. 资料 / 论文 / 竞赛 / 动态都支持占位条目
悬浮窗链接导入链路现在支持：

- 按用户选择的分类立即入库
- 占位条目状态默认为 `processing`
- 占位摘要文案会明确提示“正在抓取正文与生成摘要”
- 占位正文中会保留原始链接

也就是说，用户确认分类后，内容已经“进入库里”，而不是还停留在解析过程之外。

### 3. 后台解析完成后自动回填
本轮把链接解析链路拆成了“占位创建”和“异步回填”两个阶段。

后台解析成功时会自动回填：

- 标题
- 摘要
- Markdown 正文
- 标签
- `meta_json`
- 条目状态 `processing -> done`

后台解析失败时会自动回填：

- 失败说明
- 原始链接
- 条目状态 `processing -> failed`

### 4. 新增解析状态元数据
为了把后台解析过程呈现在 UI 中，本轮为链接条目新增了状态跟踪字段，并写入 `meta_json`：

- `parse_started_at`
- `parse_finished_at`
- `parse_feedback`

这些字段的用途是：

- 记录解析开始时间
- 记录解析结束时间
- 向前端传递当前阶段说明

### 5. 前端新增解析状态字条
WebView 主界面中的资料卡片、论文卡片和首页动态条目，现在都支持显示解析状态字条。

典型文案包括：

- `解析中 · 18秒`
- `解析中 · 1分06秒`
- `已解析 · 43秒`
- `解析失败 · 27秒`

用户不再只能看到一个静态“解析中”，而是能看到真实耗时。

### 6. 处理中条目支持阶段反馈
当后台解析仍在进行中时，资料卡片现在还能显示阶段文案，例如：

- `正在抓取网页正文`
- `网页抓取较慢，正在继续解析`
- `若正文抓取失败，将自动降级为快速链接解析`

这一步直接解决了“没有任何反馈，以为应用卡死”的问题。

### 7. 小红书 / 短链类链接加快降级
本版本顺手强化了高失败链接的体验：

- `xhslink`
- `xiaohongshu`
- `douyin`

这类链接不再盲目长时间等待通用正文抓取，而是更快降级为链接级解析，并把这一状态反馈给用户。

## 本轮解决的核心体验问题

### 修复前

- 输入链接后需要长时间等待
- 用户看不到条目是否已经加入库中
- 解析失败时缺少足够反馈
- 抓不到正文时像“卡死”

### 修复后

- 用户先选择分类
- 条目立即出现在目标分类中
- 解析过程在 UI 中持续可见
- 用户能看到解析耗时
- 失败时也会明确显示失败状态与说明

---

## 当前链路行为

以“资料”分类为例，当前流程是：

1. 输入链接  
2. 选择“资料”  
3. 立即创建一条资料占位卡片  
4. 卡片显示 `解析中 · 0秒`  
5. 后台抓取网页正文  
6. 抓取成功则结构化解析并回填  
7. 抓取失败则降级为快速链接解析  
8. 最终卡片变成 `已解析 · xx秒` 或 `解析失败 · xx秒`

## 当前版本边界

本版本解决的是“过程可见性”，不是“所有链接都能高质量抓正文”。

当前还没有完全解决的点：

- 小红书 / 抖音 / 部分短链仍然不适合稳定正文抓取
- 解析耗时目前是前端基于时间戳实时计算，并非服务端统一任务系统
- 失败条目的“重试解析”按钮尚未补上
- 资料条目的后台解析结果质量仍依赖网页可抓取程度

---

## 后续建议优先级

### P1
- 给 `processing / failed` 条目补“重试解析”
- 为资料条目增加“抓取失败，是否切 OCR 兜底”提示

### P2
- 区分“抓取阶段”和“AI 解析阶段”的状态字条
- 支持更细粒度的解析进度文案

### P3
- 引入统一后台任务模型，长期保存任务状态

---

## 版本结论

`v0.2.1` 的核心价值是把链接导入从“结果导向的黑盒等待”改成“过程可见的渐进式解析”：

- 用户先完成分类决策
- 条目立即进入目标分类
- 解析状态和耗时在 UI 中持续可见
- 即使抓不到正文，也不会再是没有反馈的死等待

后续围绕资料模块继续增强时，应以本版本的“先入库、后解析、状态可视化”交互模式为新基线。
