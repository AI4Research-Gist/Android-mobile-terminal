# v0.5.1 - 2026-04-01

## 版本主题

`v0.5.1` 是在 `v0.5.0` 知识连接版本基础上的稳定性与基础设施增强版。

这一版重点不是扩展更多新入口，而是：

- 修复主界面论文页 / 资料页的数据展示链路
- 引入同步诊断能力，便于快速定位“云端有数据但手机端不显示”的问题
- 将数据库升级策略切换为正式 migration，避免后续版本迭代再次清空本地库
- 为 `v0.6` 研究助手补上第一层基础：项目背景 Markdown 文档上传、保存与摘要

---

## 本版完成内容

### 1. WebView 主界面数据展示链路修复

本版修复了主界面论文页 / 资料页中与数据接收相关的几个关键问题：

- 修复 `receivePapers()` 中前端变量未定义导致的论文数据映射异常
- 修复前端脚本中的重复变量声明问题，避免页面直接卡在加载界面
- 主界面不再只依赖 `papers-updated / articles-updated / projects-updated` 等事件
- 新增主动获取当前列表数据的兜底接口：
  - `getPapers()`
  - `getArticles()`
  - `getCompetitions()`
  - `getInsights()`
  - `getVoiceItems()`
  - `getProjects()`

这样即使初始化事件错过，前端页面也能主动从 Android 层取回当前数据。

---

### 2. 同步诊断模式

本版新增了设置页中的“同步诊断”模块，用于直接在手机端判断：

- 当前登录用户 ID
- 当前用户名 / 邮箱
- 本地论文数
- 本地资料数
- 本地比赛数
- 本地灵感数
- 本地语音数
- 本地项目数
- 最近一次同步状态
- 最近一次同步错误
- 最近一次同步时间
- 前端实际收到的论文 / 资料 / 项目数量

这让问题排查从“猜测”变成“直接观察”。

用户价值：

- 可以快速判断问题出在账号、同步、落库还是前端接收

---

### 3. 数据库升级策略改造

本版完成了数据库升级策略的重要切换：

- 去掉 `fallbackToDestructiveMigration()`
- 改为正式 Room migration

新增：

- `DatabaseMigrations.kt`
- `MIGRATION_4_5`
  - 新建 `item_relations`
- `MIGRATION_5_6`
  - 新建 `project_context_documents`
- `DatabaseMigrations.ALL`
  - 统一注册所有 migration

同时新增迁移策略说明：

- [`DATABASE_MIGRATION_POLICY.md`](D:/Android-mobile-terminal/DATABASE_MIGRATION_POLICY.md)

这意味着：

- 后续版本升级不再默认清空本地数据库
- 原有 `items / projects / users` 可在版本升级中保留

---

### 4. OCR 摘要与显示层修正延续

本版继续收口 OCR 长文本链路：

- 继续保留 `medium_summary`
- 主界面论文 / 资料页开始优先使用中等摘要字段，而不是一味只显示 `summary_short`
- 详情页信息卡也优先使用更完整的摘要来源

目标是避免用户明明 OCR 识别到了整页文本，却只看到一段像“截断原文”的短摘要。

---

### 5. v0.6 基础设施第一步：项目背景 Markdown

本版虽然还不是完整的 `v0.6`，但已经落下第一层基础：

- 每个项目可上传一份研究背景 Markdown 文档
- 原始 Markdown 文件会保存到本地
- AI 会生成：
  - 背景标题
  - 背景摘要
  - 关键词
- 项目总览页可直接显示这份“研究背景”

这为后续：

- 项目级总结
- 灵感反查
- 多文献对比
- 项目级协作

提供了项目级长期上下文。

## 本版解决的核心问题

### 之前的问题

- 论文页在部分情况下虽然本地已有数据，但前端页面仍为空
- 页面可能卡在加载界面，难以判断是同步问题还是前端异常
- 数据库版本升级存在清空本地数据风险
- 项目级研究背景没有正式上下文入口

### 现在的状态

- 主界面接收与显示链路更稳
- 设置页可以直接看到同步诊断信息
- 数据库升级已改为正式 migration
- 项目总览页已经具备研究背景 Markdown 入口

---

## 验收建议

建议重点验收以下路径：

1. 打开设置页，查看同步诊断卡片
2. 确认当前用户 ID、本地论文数、前端收到论文数是否一致
3. 打开论文页，确认论文列表恢复显示
4. 重启 App 后再次确认不会卡在加载界面
5. 进入项目总览页，尝试上传一份研究背景 Markdown
6. 查看项目页是否显示背景摘要和关键词

---

## 版本结论

`v0.5.1` 的价值在于把 `v0.5.0` 的功能底座变得更稳、更可排查，也为 `v0.6` 的项目级研究助手铺好了第一层上下文能力。

如果说 `v0.5.0` 解决的是“条目之间开始有关系”，

那么 `v0.5.1` 解决的就是：

- 这些关系和条目能不能稳定显示出来
- 后续升级会不会把本地数据再弄丢
- 项目级上下文是否已经开始具备落地入口
