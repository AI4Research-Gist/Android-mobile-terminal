# v0.1.5 - 2026-03-21

## 版本主题
安卓端“论文索引增强 + 双语摘要 + 资料入口骨架”阶段性开发基线版本。

本版本不是一次性终版，而是把目前已经完成、已经接入、已经可继续迭代的能力整理成统一开发文档，作为后续继续开发和联调的基础。

---

## 已完成工作总结

### 1. 论文索引增强主链路
当前安卓端已经把论文条目从“标题 + 摘要”的弱结构化状态，推进到“可展示索引信息”的阶段。

已接入字段：

- `authors`
- `conference`
- `year`
- `source`
- `identifier`
- `domain_tags`
- `keywords`
- `method_tags`
- `dedup_key`
- `summary_short`
- `note`

已完成能力：

- 详情页论文索引卡
- 论文列表卡片索引展示
- `meta_json` 对象 / 字符串双兼容
- 旧记录不再因 `meta_json` 返回格式差异而丢字段

### 2. 双语摘要能力
当前双语摘要能力已经进入可用状态。

新增字段：

- `summary_zh`
- `summary_en`

当前产品策略：

- 中文优先展示
- 英文保真保留
- 列表优先展示中文短摘要
- 详情页支持 `中文 / English` 切换
- 切换偏好在本地记忆
- 详情页支持“重新生成双语摘要”

### 3. 模型与 provider 配置对齐
当前项目已完成模型配置切换，并已经在说明文档中统一口径。

当前默认配置：

- Provider：`SiliconFlow`
- 文本模型：`Qwen/Qwen3.5-397B-A17B`
- 视觉模型：`Pro/moonshotai/Kimi-K2.5`
- ASR 模型：`FunAudioLLM/SenseVoiceSmall`

说明：

- 当前项目不是直连 Qwen 官方 SDK
- 当前项目也不是直连 Moonshot / Kimi 官方 SDK
- 所有模型调用仍经由 SiliconFlow API

### 4. 资料入口（article）方向已开始落地
围绕“把资料从首页动态里解放出来”的方向，当前已完成第一轮主骨架接入。

已落地内容：

- 新增 `article` 类型
- `ArticleMeta` 结构加入领域模型
- `MainViewModel` 已增加：
  - `articles`
  - `getArticlesJson()`
- `main_ui.html` 已加入：
  - `receiveArticles`
  - `ArticlesView`
  - `articles-updated`
  - `activeTab === 'articles'`
  - 底部导航中的 `资料`
- `MainActivity` 已支持 `ARTICLE -> articles` 跳转映射
- `FloatingWindowService` 已加入“资料”分类
- `article` 详情信息卡已开始接入

### 5. 资料中的论文线索识别已开始接入
当前在文章类解析结果中，已经开始承载论文线索字段。

已接入字段：

- `referenced_links`
- `paper_candidates`

当前候选识别类型包括：

- `arxiv`
- `doi`
- `pdf`
- `paper_page`

说明：

- 当前仍处于“识别与展示准备阶段”
- 尚未进入“一键导入论文”的工作流

## 当前开发基线

### 论文方向

- 手机端快速采集
- AI 补全索引
- 中文优先展示
- 英文保真保留

- 资料需要独立入口
- 不应继续只作为首页动态
- 文章中的论文线索应先识别出来
- 一键导入论文留到后续版本
