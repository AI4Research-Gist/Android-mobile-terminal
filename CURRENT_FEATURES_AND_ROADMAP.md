# Current Features And Roadmap

## 1. 文档定位

本文档用于汇总 `Gist / AI4Research` 当前已经实现的能力，以及基于现有架构最适合继续推进的方向。

判断原则：

- “已实现”指仓库当前代码和现有文档已经能对应上的能力
- “可继续实现”指基于当前 `Room + NocoDB + SiliconFlow + WebView/Compose Hybrid` 架构，可以顺延落地的能力
- “后续方向”指值得继续投资，但可能需要更多产品设计、数据结构或工程化支持的方向

当前项目已经进入 `v0.6.0` 的第一阶段落地，`项目级 AI 总结` 与 `灵感反查相关资料` 已经可用，但多文献对比、导出和周期性研究回顾仍属于后续迭代范围。

---

## 2. 当前已经实现的功能

### 2.1 采集与导入

- 支持链接导入，并由 AI 分类解析为 `paper / article / competition / insight / voice`
- 支持悬浮窗助手，包括全屏截图、区域截图、剪贴板链接检测、手动输入链接
- 支持 OCR 图像识别与长文本整理
- 支持独立语音录制页，完成录音、ASR 与 AI 优化
- 支持在灵感条目中附加原始语音和图片

### 2.2 条目管理与浏览

- 统一管理 `paper / article / competition / insight / voice` 五类研究条目
- 支持搜索、筛选、项目归属、星标、已读状态管理
- 资料页和论文页支持按项目、年份、来源、作者、关键词、标签等维度筛选
- 首页已收敛为“灵感”页，支持页面内新建、搜索、显示已读、隐藏已读
- 资料条目支持本地优先显示，占位条目可先展示，再异步补全远端解析结果
- 链接解析状态支持前端可视化，用户能看到解析中、解析完成、失败和耗时

### 2.3 详情页能力

- Compose 详情页支持 Markdown 渲染与编辑保存
- 支持条目标记已读、星标、删除、项目归属变更
- 灵感详情支持图片预览与原始语音播放
- 详情页支持围绕当前条目发起 AI 问答
- `paper / article` 支持结构化阅读卡

### 2.4 AI 解析与辅助

- SiliconFlow 已用于链接解析、OCR、摘要生成、语音优化
- OCR 长文本链路已增加 `medium_summary`，显示层优先使用中等摘要
- 条目详情页 AI 问答会优先参考 OCR 原文和摘要上下文
- 支持重新生成双语摘要
- 支持为 `paper / article` 生成结构化阅读卡草稿

### 2.5 知识连接与项目视角

- 支持 `paper` 去重分组
- 支持 `article -> paper` 自动关联
- 支持 `insight -> item` 手动关联
- 项目页已经不只是筛选维度，而是具备项目总览能力
- 项目总览页支持：
  - 项目概况
  - 最近新增条目
  - 重点论文
  - 灵感汇总
  - 关系统计

### 2.6 项目研究背景上下文

- 每个项目支持上传一份 Markdown 研究背景文档
- 原始 Markdown 会落到本地文件系统
- AI 会提取研究背景标题、摘要、关键词
- 项目总览页已经可以直接展示研究背景摘要与关键词

这部分是 `v0.6` 的第一块核心基础，因为后续项目级 AI 总结、灵感反查、多文献对比都可以复用这份上下文。

### 2.7 项目级研究助手

- 项目总览页支持生成 `AI 项目总结`
- 总结结果包含：
  - 当前主题
  - 最近进展
  - 关键文献
  - 灵感焦点
  - 待补问题
  - 下一步建议

### 2.8 灵感反查相关资料

- 灵感详情页支持 `反查相关资料`
- 系统会结合灵感正文、项目归属、项目背景摘要和本地候选资料做推荐
- 结果包含推荐条目、推荐理由和可继续追问的问题
- 支持一键采纳推荐结果为显式关联

### 2.9 账号、同步与存储

- 已接入基于 NocoDB 用户表的注册与登录
- 本地使用 `EncryptedSharedPreferences` 保存 token
- 当前条目与项目已按登录用户进行隔离
- 设置页已支持手动创建项目
- 设置页支持同步诊断，可查看：
  - 当前用户 ID
  - 本地条目数量
  - 未上云条目数
  - 可补传条目数
  - 本地处理中条目数
  - 最近同步状态
  - 最近同步错误
  - 前端实际接收数量
- Room 已作为本地 SSOT
- 数据库升级已切换为正式 migration，不再默认清空本地库
- 刷新时会自动尝试补传本地已完成但尚未上云的条目

### 2.10 界面与工程基础

- 项目采用 Compose + WebView Hybrid 架构
- 主界面和登录页为 WebView 内 React/Tailwind 页面
- 详情页、项目页、语音页为原生 Compose
- 支持 WebView 预热与缓存
- Splash 会等待初始化完成，提升启动体验
- 图片 OCR / 视觉识别链路已增加更长超时和缩图重试策略

## 3. 基于现有架构，下一阶段最适合继续实现的功能

以下能力与当前项目的数据结构、AI 服务和页面组织最连续，投入产出比最高。

### 3.1 多文献对比

可实现内容：

- 先支持两篇 `paper/article` 对比
- 在条目详情页或列表页增加“加入对比”
- 输出相同点、差异点、互补点、是否适合当前项目
- 引用已有结构化阅读卡字段，减少纯自然语言拼接带来的空泛感
- 生成单独的对比结果页或 Markdown 结果

为什么现在适合做：

- 条目详情已有 AI 对话与结构化阅读卡
- 项目背景摘要可作为对比时的偏好上下文

### 3.2 周回顾 / 未读积压 / 即将截止提醒

可实现内容：

- 统计本周新增条目
- 统计本周重点论文与新增灵感
- 汇总未读论文、未读资料、长时间未处理条目
- 汇总近期比赛截止时间
- 生成可读的研究节奏回顾卡片

为什么现在适合做：

- 当前已经有条目时间、已读状态、比赛数据和项目归属
- 这类能力更偏读取聚合，不需要重构底层

### 3.3 Markdown / 周报 / 文献清单导出

可实现内容：

- 导出项目总览 Markdown
- 导出某项目的本周回顾
- 导出重点文献列表
- 导出某个对比结果或某次 AI 项目总结

为什么现在适合做：

- 当前系统内部已经有较完整的结构化字段
- Markdown 是最适合现阶段的低成本导出格式

## 4. 后续值得继续完善的方向

这些方向并不是不能做，而是比上一节更偏“增强型能力”或“产品化能力”。

### 4.1 知识连接增强

- 从强规则匹配升级到“强规则 + 弱匹配建议”
- 增加基于标题、作者、年份的弱匹配候选
- 增加 AI 辅助的关系建议和人工确认 UI
- 为关系增加更清晰的置信度、来源说明和人工编辑入口

### 4.2 项目上下文真正参与所有 AI 调用

- 目前研究背景 Markdown 已经能上传和摘要，但还没有系统性接入所有 AI 场景
- 后续可以让它统一参与：
  - 项目总结
  - 双文献对比
  - 灵感反查
  - 条目重分析
  - 问答回答增强

这会让产品从“会分析单条内容”更明显地升级到“理解你在做什么研究”。

### 4.3 可追溯性与可信度增强

- 为 AI 输出补“依据来源”
- 标记回答参考了哪些条目、哪些摘要、哪些项目背景片段
- 避免总结结果看起来像模板化空话
- 提供“重新生成”和“查看原始依据”双入口

### 4.4 解析与采集链路继续补强

- 资料模块增加手动 OCR 兜底入口
- 提升公众号、网页、长图、多图 OCR 的稳定性
- 细化解析失败原因和重试策略
- 增强后台解析任务的状态机与异常恢复

### 4.5 同步与离线体验增强

- 强化本地缓存与断网可读能力
- 优化同步冲突策略
- 增加更完整的重试、补偿、失败队列
- 减少 Web UI 对运行时 CDN 的依赖，逐步转向更可控的本地打包资源

### 4.6 安全性与产品化

- 当前仍偏直连式 NocoDB 架构，后续建议增加服务端代理层
- 强化认证与权限边界
- 收敛明文 HTTP 与敏感配置暴露风险
- 为正式产品化准备更稳定的云端数据模型和接口边界

---

## 5. 推荐的开发优先级

如果按“价值高、复用现有能力多、落地风险相对低”的标准排序，建议优先做：

1. 双文献对比
2. 周回顾 / 未读积压 / 即将截止提醒
3. Markdown / 周报 / 文献清单导出
4. 知识连接增强
5. 同步、离线与产品化安全能力

---

## 6. 一句话判断当前项目所处阶段

当前项目已经完成了“科研资料采集器 + 条目管理器 + 条目级 AI 助手 + 第一阶段项目级研究助手”的主体建设。

下一阶段最关键的，是把已经上线的项目总结与灵感反查继续往“对比、导出、回顾”三条高价值链路扩展。
